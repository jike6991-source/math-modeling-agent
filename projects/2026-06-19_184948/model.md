# 展销会临时工排班优化模型

## 1. 模型假设
- 所有临时工能力相同，均可胜任任何小组工作。
- 需求表中每小时需求人数为在该小时必须在岗的临时工数量，覆盖需求即可，不考虑工作交接。
- 临时工每天工作时间严格划分为若干连续的1小时时段，8:00-9:00为时段0，...，18:00-19:00为时段10，共11个时段。
- 每名临时工每天必须且只能选择一种工作模式（含休息），工作模式由连续8小时构成（问题1、2）或由两个4小时段构成（问题3）。
- 每名临时工在10天内必须恰好工作8天、休息2天；工作日内不得缺勤。
- 问题1中临时工10天内固定服务某一小组；问题2中每天可换组但一天内只能服务于一个小组；问题3中每天可换组且一天内可服务于至多两个小组（每个小组连续4小时，两段间至少休息2小时）。

## 2. 符号说明
- `D = 10`：总天数，索引 `d = 0,…,9`
- `H = 11`：每天工作小时数（8:00~19:00），时段索引 `h = 0,…,10`
- `G = 10`：小组数，索引 `g = 0,…,9`
- `M`：每日所有可行工作模式集合（含休息模式 `m=0`）
  - 问题1、2：连续8小时模式，共 `4×G = 40` 种（开始时段 0,1,2,3 × 10 组），加上模式0（休息）共 41 种
  - 问题3：连续8小时模式（40 种）+ 拆分模式（两段4小时，时段组合3种 × 小组对 10×10 = 300 种），共 340 种，加休息模式共 341 种
- `a_{d,h,g,m}`：参数，若工人在第 `d` 天采用模式 `m` 会在第 `h` 时段覆盖小组 `g`，则值为1，否则0
- `demand[d][h][g]`：第 `d` 天第 `h` 时段小组 `g` 所需临时工人数（整数）
- `K`：临时工总人数上界估计值（可通过启发式获得，如各天各小时总需求的 max 再乘一个系数）
- 决策变量：
  - 问题通用：
    - `z_i ∈ {0,1}`：工人 `i` 是否被使用（`i=1,…,K`）
    - `x_{i,d,m} ∈ {0,1}`：工人 `i` 在第 `d` 天是否采用模式 `m`（`m=0` 表示休息）
  - 问题1额外：
    - `y_{i,g} ∈ {0,1}`：工人 `i` 是否固定分配至小组 `g`
- 目标：最小化使用的临时工总数 `∑_{i=1}^{K} z_i`
- 约束条件：
  - 每个工人每天只能选择一种模式：`∑_{m∈M} x_{i,d,m} = 1, ∀i, d`
  - 工作天数约束：`∑_{d=0}^{D-1} ∑_{m∈M, m≠0} x_{i,d,m} = 8·z_i, ∀i`  （若使用则必须工作8天，否则全休息）
  - 需求覆盖约束：`∑_{i=1}^{K} ∑_{m∈M} a_{d,h,g,m} · x_{i,d,m} ≥ demand[d][h][g], ∀d,h,g`
  - 问题1小组固定约束：`∑_{g} y_{i,g} = z_i, ∀i` 且 `x_{i,d,m} ≤ y_{i,g(m)}, ∀i,d,m` （其中 `g(m)` 表示模式 `m` 涉及的小组；对问题1，每个工作模式 `m` 指定唯一小组）
  - 问题2：无额外固定小组约束，`x_{i,d,m}` 可自由选择不同小组的模式
  - 问题3：模式集 `M` 内已包含所有合法拆分模式，无需额外约束

## 3. 问题1、2、3的整数线性规划模型
**主问题（集合覆盖/指派）**
采用以单个临时工为索引的指派型整数规划，上界 `K` 取足够大的值（如各天小时总需求的最大值×1.5），通过求解器直接最小化实际使用人数。

**模式集合的生成**
- 问题1和问题2（连续8小时）：
  枚举开始时段 `s ∈ {0,1,2,3}`，时段覆盖 `h ∈ [s, s+7]`。对于小组 `g`，模式 `m` 标记为 `(s,g)`，`a_{d,h,g,m}=1` 当且仅当该工人当天选择该小组且时段 `h` 被覆盖。
  休息模式 `m=0` 对应 `a=0` 对所有 `(h,g)`。
- 问题3（可拆分）：
  除上述40种连续模式外，增加拆分模式：
  枚举第一段开始 `s1 ∈ {0,1}`，第二段开始 `s2 ∈ {s1+6, s1+7}` 且 `s2 ≤ 7`；枚举两个小组 `g1,g2 ∈ [0,9]`（允许 `g1=g2`）。模式覆盖时段集合为第一段 `[s1, s1+3]` 和第二段 `[s2, s2+3]` 分别作用于 `g1` 和 `g2`。
  注意：如果 `g1=g2`，则该工人当天在该小组工作两段共8小时（中间休息至少2小时），与连续8小时模式有重叠但可以保留作为可选模式。

**求解策略**
由于模型规模较大，采用 PuLP 调用 CBC 求解器，并设置时间限制（每个子问题 60 秒）。若求解器达到时间上限，返回当前最佳整数可行解。求解后，统计实际使用的工人数即为目标值，同时提取非零变量的 `x` 值作为排班方案。

## 4. 模型求解思路
1. 数据预处理：从 Excel 读取需求矩阵 `demand[10][11][10]`。
2. 根据三个问题分别构建模式集合和参数矩阵 `a`。
3. 对每个问题，先估计一个合理工人数上限 `K`（基于总需求最大值），建立 MIP 模型，设置目标函数与约束，调用求解器。
4. 记录求解状态、最优人数、运行时间，并提取排班方案。
5. 进行约束满足性验证：检查覆盖是否全部满足，个人工作天数是否为8。
6. 生成至少5张可视化图表并保存。
